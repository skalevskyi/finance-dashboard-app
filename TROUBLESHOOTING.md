# Troubleshooting Guide

## Якщо проєкт не працює

### 1. Перевірте консоль браузера
Відкрийте DevTools (F12) і перевірте вкладку Console на наявність помилок.

### 2. Перевірте, чи запущений dev сервер
```bash
npm run dev
```
Сервер повинен бути доступний за адресою `http://localhost:5173` або `http://localhost:5174`

### 3. Перевірте залежності
```bash
npm install
```

### 4. Очистіть кеш
```bash
# Видаліть node_modules та package-lock.json
rm -rf node_modules package-lock.json
# Або на Windows:
rmdir /s node_modules
del package-lock.json

# Перевстановіть залежності
npm install
```

### 5. Перевірте версію Node.js
Проєкт потребує Node.js 20.19+ або 22.12+
```bash
node --version
```

### 6. Перевірте збірку
```bash
npm run build
```
Якщо збірка не вдається, перевірте помилки.

### 7. Типові проблеми

#### Білий екран
- Перевірте консоль браузера на помилки
- Перевірте, чи правильно імпортуються компоненти
- Перевірте, чи існують всі файли перекладів

#### Помилки з i18n
- Перевірте, чи всі JSON файли в `src/i18n/en/` та `src/i18n/ua/` існують
- Перевірте синтаксис JSON файлів

#### Помилки з темою
- Перевірте, чи правильно працює localStorage
- Перевірте консоль на помилки з useThemeStore

#### Помилки з роутингом
- Перевірте, чи всі компоненти імпортуються правильно
- Перевірте шляхи в `src/app/router.tsx`

### 8. Якщо нічого не допомагає

1. Перевірте логи в терміналі, де запущений `npm run dev`
2. Перевірте Network вкладку в DevTools на наявність помилок завантаження
3. Спробуйте відкрити проєкт в іншому браузері
4. Перевірте, чи не блокує браузер якісь розширення

### 9. Контакт
Якщо проблема залишається, перевірте:
- Версію Node.js
- Версію npm
- Чи правильно налаштовані шляхи (@/ alias)
- Чи всі файли створені правильно
